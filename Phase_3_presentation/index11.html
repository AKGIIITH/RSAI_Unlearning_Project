Challenges & Future Work

Current Limitations & Path Forward

Current Challenges

Low SAE Expansion Ratio

Only 4× expansion (1024 → 4096). State-of-art SAEs use 16×–64× for proper monosemanticity. Features remain polysemantic — multiple concepts entangled in single dimensions.

Insufficient Training Data

Only ~10K images due to Kaggle GPU memory (P100, 16 GB). Ideal SAE training requires 100K+ diverse images for robust feature decomposition.

Too Few Epochs

Only 2 training epochs. SAE convergence typically needs 10–50 epochs. Features are undertrained and noisy.

Language Leakage Unaddressed

Vision-side clamping alone cannot fix text-decoder knowledge leakage (LL=96%). The LLM backbone independently encodes "zebra" knowledge in its weights.

Matryoshka SAE Incompatible

Planned to use pretrained Matryoshka SAE from recent literature, but its compression layer doesn't exist in LLaVA-1.5-7B's CLIP encoder. Had to train from scratch.

Future Improvements

Better SAE Architecture

Increase to 16× expansion (1024 → 16384), train for 20+ epochs on 100K+ images, add decoder bias tying, and use TopK activation instead of L1 for sharper sparsity.

Dual-Path Unlearning

Combine vision-side SAE clamping with language-side interventions (e.g., token probability suppression for "zebra", "stripes") to address language leakage jointly.

Improved Evaluation

Switch from strict string match to semantic similarity (e.g., BERTScore) for RA evaluation. Add more diverse retain concepts beyond horse/donkey.

Alternative: Change Model

If SAE quality remains poor, consider switching to a model with compatible pretrained SAEs, or use a smaller VLM where SAE training is more feasible.
